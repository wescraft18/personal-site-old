I can ride my Enumerable#cycle

Joey S. - 6/7/14

When I first ran the cycle method it looped my array forever and I thought, "why would anyone ever need this?" Then i saw that you can pass a parameter for how many times to run the cycle and I definitely saw some good use of this.

The cycle enumerable is great if you are trying to infintely iterate over an array. It can clean up your code and provide a much simplier solution to this issue.

>> new_array = [1, 2, 3]
>> 3.times { |x| p new_array[x % new_array.size] }
1
2
3

What this is doing is iterating each item of the array, for a total of 3 times, and print that to the screen. Now if we implement the Enumerable#cycle onto this same feature, we can do this much simplier with a cleaner block.

>> new_array = [1, 2, 3]
>> new_array.cycle(1) { |x| puts x }
1
2
3

This will give us the same thing, as it made 1 cycle through the array and put each result. The times operator seems better, however, as this will always make one complete cycle–but there is a way around that as well and lives in the Enumerator#next.

>> new_array = [1, 2, 3]
>> new_array = new_array.cycle
>> new_array.next
1
>> new_array.next
2

This way you can select any element in the array while it is looping through the array.
