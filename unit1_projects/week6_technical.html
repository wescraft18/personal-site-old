To the front of the CLASS

Joey S. - 6/19/14

Class methods are called on a class. This is the major difference between a class method and an instance method. To run a method from a class method you will need to run the class on the method, as depicted below.

class SweetClass
  def self.class_method
   puts "This method was run as a class method."
  end

  def instance_method
   puts "This method was run from an instance."
  end
end


instance = SweetClass.new
SweetClass.class_method
SweetClass.instance_method
>>"This method was run as a class method."
>>NoMethodError: undefined method 'instance_method' for SweetClass:Class

The method that assigns the class to itself returns the proper input, but the instance_method method, which is run within the class, comes up with a NoMethodError as new instance was created of the class, well, it was created but it didn't get called.

class SweetClass
  def self.class_method
   puts "This method was run as a class method."
  end

  def instance_method
   puts "This method was run from an instance."
  end
end

------------------------------
instance = SweetClass.new
SweetClass.class_method
SweetClass.instance_method
>>"This method was run as a class method."
>>NoMethodError: undefined method 'instance_method' for SweetClass:Class

------------------------------
instance.instance_method
instance.class_method
>>"This method was run from an instance."
>>NoMethodError: undefined method class_method for #SweetClass:30303

So running the instance on a method in the class returns the proper response, but running the instance on the class method returns a NoMethodError that we saw before. This is, according to a lot of sites, something heavily used in Rails for ActiveRecord. But more on that later, I'm sure.

Consensus for class method use was to shorten code and remove initialization of variables. Also, not having to create a state for each method, just passing self instead, saves a lot of time and cleans up code. I haven't really had the opportunity to use a class method and have primarily worked in instance methods, but I will definitely be trying now!
